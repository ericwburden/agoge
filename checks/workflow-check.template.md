# Workflow Check Template

## Name

Provide a short name for the check or rubric.

## Purpose

Describe what this check is meant to validate.

## Applies To

- Describe which workflows, artifacts, or delivery stages this check applies to.
- Note when this check should not be used.

## Criteria

- List the specific criteria being evaluated.
- Phrase them so different reviewers or agents would judge them similarly.

## Scoring Or Outcome

- Describe whether this is pass/fail, scored, or advisory.
- Explain what constitutes an acceptable result.

## Evidence Required

- List the artifacts, logs, or outputs needed to perform the check.
- State what to do if evidence is missing.

## Failure Response

- Describe what should happen when the check fails.
- Note whether work should stop, be reworked, or be escalated.

Use this section for examples, calibration guidance, or links to related standards and workflows.
