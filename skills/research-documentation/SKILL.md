# Research & Documentation

Pull relevant local sources, synthesize findings, and publish clear Markdown briefs or reports with explicit source grounding.

## Quick start
1) Identify the question, decision, or documentation goal.
2) Find relevant local sources such as notes, artifacts, specs, meeting outputs, or supporting documents.
3) Choose an output shape: quick brief, summary, comparison, or comprehensive report.
4) Synthesize the material into a local Markdown document with clear sections and citations or source references.
5) Finalize with highlights, risks, and open questions.

## Workflow

### 1) Gather sources
- Search local Markdown files first.
- Prefer repo artifacts, notes, workflows, specs, and meeting outputs before reaching for external material.
- Track source file paths and relevant sections so claims can be traced back later.

### 2) Select the format
- Quick readout → quick brief.
- Single-topic investigation → summary.
- Option tradeoffs → comparison.
- Deep dive or decision support → comprehensive report.

### 3) Synthesize
- Outline before writing.
- Group findings by themes, questions, or decisions.
- Flag contradictions, missing evidence, or stale inputs rather than smoothing them over.
- Keep the user goal in view: decision, summary, recommendation, or handoff.

### 4) Produce the document
- Draft the output in local Markdown.
- Include title, summary, key findings, supporting evidence, and recommendations or next steps when relevant.
- Add explicit source references using file paths, filenames, or other durable local citations.

### 5) Finalize
- Add highlights, risks, assumptions, and open questions.
- If follow-up work is needed, point to the next artifact or workflow rather than mixing planning into the research summary.

## Notes

- This is a local-Markdown adaptation of a Notion-based research-and-documentation pattern.
- Prefer local files and artifacts as the default research base.
- For Business Analyst work, this skill is best used to support business objectives, process analysis, or requirement clarification when multiple source files must be synthesized.
