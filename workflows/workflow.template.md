# Workflow Template

## Name

Provide a short, stable workflow name.

## Purpose

Describe the outcome this workflow is meant to produce.

## When To Use

- List the trigger conditions for this workflow.
- Describe the kinds of tasks or requests that should route here.

## Inputs

- Required files, specs, or context.
- Optional supporting materials.

## Outputs

- Primary artifact this workflow should produce.
- Secondary artifacts, notes, or validation evidence.

## Skills And Tools

- List the skills involved.
- List the tools or scripts involved.
- Note why each part is included.

## Sequence

1. Describe the first step and what it should produce.
2. Describe the next step and the handoff artifact it consumes.
3. Continue until the workflow reaches a validated outcome.

## Decision Points

- Describe branch conditions in the workflow.
- Explain what should happen when assumptions fail.

## Validation

- List the checks that confirm the workflow succeeded.
- Define what evidence should be captured.

## Failure Handling

- Describe fallback behavior.
- Describe when to stop rather than continue.

## Notes

Use this section for naming conventions, examples, or implementation details that do not belong in the core sequence.
