# Orpheum 1.2.0 Release Notes

## Summary

Orpheum 1.2.0 adds an explicit session-close lifecycle so finalized work can be archived safely and projects can transition cleanly into the next scenario without manual `.orpheum/` cleanup.

## Highlights

- Added `orpheum session close` to archive finalized sessions into `.orpheum-archive/`.
- `orpheum status --json` now reports whether cleanup is safe, why it is or is not ready, and which command should be run next.
- `orpheum scenario apply` now gives better `SESSION_ALREADY_ACTIVE` guidance and supports `--archive-current` to transition directly from a finalized session into a new scenario.
- Updated the installed project skill and agent-facing docs so agents are directed toward the safe lifecycle path instead of manual control-dir edits.

## Upgrade Notes

- Finalized sessions should now be closed with `orpheum session close --json` before starting a new scenario, unless you intentionally use `orpheum scenario apply <id> --archive-current --json`.
- Consumers that inspect `orpheum status --json` can now rely on `cleanup_ready`, `cleanup_reason`, and `recommended_next_command`.
- Active-session apply failures now include more transition guidance, which may be useful for host integrations that surface CLI errors directly.

## Validation

- `cargo test -p orpheum-core`
- `cargo test -p orpheum --test cli`
