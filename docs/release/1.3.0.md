# Orpheum 1.3.0 Release Notes

## Summary

Orpheum 1.3.0 adds the missing explicit finalization step between successful checks and safe session close so the CLI lifecycle now matches the actual scenario-completion workflow.

## Highlights

- Added `orpheum session finalize` as the explicit transition from active scenario work into finalized session state.
- `orpheum status --json` now reports `finalize_ready` and `finalize_reason`, and recommends `orpheum session finalize --json` when checks pass but the session is still active.
- Finalizing a session now promotes remaining pending workflows into completed workflows and moves the session phase to `session-finalized`.
- Updated the installed local Orpheum skill, consumer docs, and README so agents are guided through the full lifecycle: check, finalize, close, then apply the next scenario.

## Upgrade Notes

- The recommended lifecycle is now `orpheum check run --json`, then `orpheum session finalize --json`, then `orpheum session close --json`.
- Consumers that inspect `orpheum status --json` can now rely on `finalize_ready`, `finalize_reason`, `cleanup_ready`, and `recommended_next_command` to drive the next lifecycle action.
- Existing finalized-session close behavior is unchanged; this release fills the lifecycle gap before close.
